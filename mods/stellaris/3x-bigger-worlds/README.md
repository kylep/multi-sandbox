# 3x Bigger Worlds

A Stellaris mod for **Pegasus v4.4.x** that triples every dimension of the
"resource real estate" on a world:

1. **3x planet & moon size.** Habitable planets spawn at sizes 36–75 instead
   of 12–25. Habitable moons spawn at 30–45 instead of 10–15. Because vanilla
   gives 1 district per planet size, every habitable world has ~3x as many
   district slots from the moment of generation.
2. **3x deposit max-district adds.** Every planetary feature that grants
   `district_mining_max_add = N`, `district_farming_max_add = N`,
   `district_generator_max_add = N`, etc. now grants `N * 3`. Same for
   habitat deposits (`district_hab_*_max_add`) and the
   `planet_max_districts_add` modifier (used by oversized-deposit features
   like Crystalline Highlands).
3. **3x orbital deposit yields.** Every `produces = { minerals = N }`,
   `energy = N`, `food = N`, `alloys = N`, `research = N`, `exotic_gases = N`,
   etc. on uncolonizable bodies is multiplied by 3.
4. **3x strategic resource district contribution.** The
   `@DISTRICTS_FROM_SR_DEPOSITS` and `_SMALL` variables that drive zone
   district counts are also tripled.

**Untouched intentionally:**

- **Negative values.** Blocker penalties (e.g. `district_farming_max_add = -1`
  on a swamp blocker, `planet_max_districts_add = -2` on a wilderness blocker)
  stay at vanilla magnitude. Clearing a blocker still reclaims roughly the
  same number of slots — on a planet that's 3x larger, blockers become
  proportionally less painful, which is the intended feel.
- **`upkeep = { … }` blocks.** Vanilla magnitude. A deposit that vanilla
  defined as `produces = { energy = 4 } upkeep = { energy = 1 }` becomes
  `produces = { energy = 12 } upkeep = { energy = 1 }` — net energy is +11
  instead of vanilla +3 (a ~4× swing on that axis, not 3×). Few deposits use
  upkeep in vanilla so the aggregate impact is small; symmetric scaling
  would be a separate behaviour change.
- **`@VAR` references in deposit files.** Lines like
  `district_farming_max_add = @DISTRICTS_FROM_SR_DEPOSITS` resolve to the
  *tripled* value via the scripted_variables override — they're not rewritten
  in place, but the runtime result is still 3x.
- **Visual planet rendering.** This is a gameplay-only mod. Planet entities
  in the system view stay vanilla-sized.

## How it works

Two override surfaces:

- `mod/common/scripted_variables/zz_bworlds_planet_sizes.txt` — redefines the
  six size / SR-deposit variables. The `zz_` filename prefix puts it after
  vanilla's `100_scripted_variables_zones.txt` in alphabetical load order, so
  our values win.
- `mod/common/deposits/*.txt` — same-named overrides for all 24 vanilla
  deposit files. Generated by `scripts/build.py` from the locally installed
  vanilla files. Every positive `district_*_max_add` and every numeric value
  inside a `produces = { ... }` block is multiplied by 3; negatives and
  `@VAR` references are left untouched.

When Stellaris patches, re-run `scripts/build.py` and the same regex
transformations re-apply to the new vanilla files. Behaviour is patch-resilient
by construction.

## Build

```bash
python3 scripts/build.py
```

Reads from `~/Library/Application Support/Steam/steamapps/common/Stellaris/common/`.
Edit `VANILLA_ROOT` in `scripts/build.py` if your install is elsewhere.

The multiplier is a constant at the top of `build.py` (`MULTIPLIER = 3`). To
build a 2x or 5x variant, edit that and re-run.

## Pre-game verification

```bash
python3 scripts/preflight.py
```

Nine automated checks — brace balance, vanilla key coverage on every deposit
file, scripted_variables completeness, exact-multiplier verification on every
generated value, negative-value preservation, localisation BOM, descriptor
sanity, deploy symlink integrity, and thumbnail validity (real PNG bytes,
512×512, no alpha channel, < 1 MB — verified against the parsed IHDR, not the
filename, after the first Multi-Megastructures Workshop upload broke on a
JPEG-with-`.png`-extension). Full table in [`TESTING.md`](TESTING.md).

## Deploy locally

```bash
scripts/deploy.sh
```

Creates:

- `~/Documents/Paradox Interactive/Stellaris/mod/3x-bigger-worlds` → symlink to
  `apps/mods/stellaris/3x-bigger-worlds/mod`
- `~/Documents/Paradox Interactive/Stellaris/mod/3x-bigger-worlds.mod` →
  launcher descriptor with absolute `path=`

Re-running `scripts/build.py` is picked up immediately by the launcher — no
need to re-deploy.

## Thumbnail

The shipped `mod/thumbnail.png` was generated by `scripts/generate-thumbnail.mjs` (Gemini 2.5 Flash Image, via `GEMINI_API_KEY` from the environment) and canonicalized via `scripts/process-thumbnail.sh`. To regenerate:

```bash
GEMINI_API_KEY=... node scripts/generate-thumbnail.mjs   # writes mod/thumbnail-src.png (1024x1024 raw)
scripts/process-thumbnail.sh                              # re-encodes to mod/thumbnail.png + syncs blog image
python3 scripts/preflight.py                              # C9 enforces real-PNG / 512x512 / no-alpha / <1MB
```

To swap in a hand-drawn or externally-generated image instead, just drop it at `mod/thumbnail-src.<ext>` (PNG / JPEG / WebP / TIFF — any format `sips` understands) and run `scripts/process-thumbnail.sh`. The pipeline is `sips`-based and macOS-only; on Linux substitute `convert` (ImageMagick) or `ffmpeg`.

Override `IMAGE_MODEL` (e.g. `IMAGE_MODEL=gemini-3-pro`) for a different Gemini image model. The Gemini API call lives directly in `generate-thumbnail.mjs` — no external Node deps required.

## Test in-game

1. Launch Stellaris → Paradox Launcher → Mods → enable **3x Bigger Worlds** → Play.
2. Start a new game.
3. **Feature 1 check (planet size).** Survey the home system. The home planet
   should be size 75 (vanilla starting capital is size 25). Other habitable
   planets in nearby systems should be in the 36–75 range.
4. **Feature 2 check (deposit max-district adds).** Land on the home planet
   and inspect the planetary features panel. A vanilla "Mineral Fields"
   feature grants +1 max mining district; under this mod it grants +3.
5. **Feature 3 check (orbital deposit yields).** Find an asteroid or barren
   world with a mineral deposit shown in the system view. A vanilla
   "Minerals 2" deposit yields 2 minerals per month; under this mod it yields 6.
6. Tail `~/Documents/Paradox Interactive/Stellaris/logs/error.log` and check
   for unresolved variable references or parse errors after starting the game.

## Known caveats

- **Balance.** 3x resources will trivialize the early-mid game. Pop growth,
  upkeep, and AI weights are untouched, so empires will outgrow their pop
  base. This is intentional for a "fewer, bigger, richer worlds" feel — if
  you want a stricter balance, drop `MULTIPLIER` to 2 in `build.py`.
- **AI.** Vanilla AI weights for district / building construction assume
  vanilla sizes. AI empires will still build, but may under-utilize the new
  slots.
- **Save compatibility.** Planet size changes only affect *newly generated*
  galaxies, not existing saves. Deposit yield changes apply immediately on
  load (modifiers re-evaluate).
- **Mod conflicts.** Any other mod that overrides `common/deposits/*.txt` or
  redefines the same `@habitable_planet_*_size` variables will conflict —
  last-loaded wins. Compatible with mods that only touch
  `common/planet_classes/`, `common/districts/`, `common/buildings/`, etc.
- **Crash risk.** Stellaris does not crash at size 75 in v4.3 testing, but
  some scripted events that hard-code `planet_size <= 25` conditions may
  behave oddly. Sample with a quick game before committing to a long run.
